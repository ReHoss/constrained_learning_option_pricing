# Neural DDE based control

## Moment-closure (5-moment) and differentiable solvers

This repository also contains a self-contained implementation of:

- **Dynamic Regularization Manifold (DRM) for the 5-moment \(\beta\)-closure** (Schaerer & Torrilhon, 2015), where the static scalar \(\beta\) is replaced by a neural network \(\beta_\theta(\hat Q,\hat R)>0\) and trained with a differentiable hyperbolicity (spectral) penalty.
- **Two 1D PyTorch solvers** designed for closure learning:
  - **Solver I (kinetic oracle)**: BGK discrete-velocity method to generate “exact” moment data.
  - **Solver II (macroscopic solver)**: 5-moment balance law with Minmod reconstruction, HLL fluxes (Harten–Lax–van Leer), and Heun (RK2) time stepping, with a pluggable closure function and gradient flow through flux computations.

Docs:

- **DRM \(\beta\)-closure (math + training loop)**: `documents/methodology/dynamic_regularization_manifold_beta_closure.md`
- **Numerical solvers (math + architecture)**: `documents/methodology/numerical_solvers_architecture.md`

Optional dependency note:

- PyTorch is treated as optional in this repo; install the ML extras if needed (see `pyproject.toml`).

## Installation

Create your favorite virtual environment, clone this repository with `git clone`.
Change directory to the cloned repository
```
cd learning_moments_kinetics/
```
then install the package with
```
pip install -e .
```

Note, it is important to install the package in editable mode (`-e` option), so that generated data is written under the repository `data/` directory.


## Structure overview

- `bash_scripts/` contains bash scripts to run tools such as `tensorboard` and `mlflow`.
- `config/` contains configuration files for the experiments.
- `control_dde/` contains the main codebase.
- `learning_moments_kinetics/` contains the moment-closure code:
  - `learning_moments_kinetics/closures/`: dynamic \(\beta\)-closure (`beta_closure_net.py`) and related utilities
  - `learning_moments_kinetics/solvers/`: 1D BGK kinetic solver + 5-moment macroscopic solver + stationary shock-structure ODE solver + 6-axiom shock validation suite
  - `learning_moments_kinetics/visualization/`: shock validation figure generator (`shock_visualization.py`)
- `data/` contains data generated by the experiments and example data.
- `documents/methodology/` contains methodology notes and math documentation for the closure and solvers.
- `experiments/` contains example scripts to run the experiments.
- `oderl/` reference code of *Yildiz et al. (2020)*.
- `torchbnn/` repository for Bayesian neural networks.
- `test` contains tests for the codebase.

## Output Organization (Scientific Runs)
When scripts generate artifacts, they should write them into a timestamped “run directory”:
`data/<script_name>/<timestamp>/`

Each run directory is expected to contain:
- `run.log`: logging output produced by the script
- `command.txt`: the exact command line used (best-effort: `sys.argv`)
- `run_metadata.json`: a metadata bundle with parameters, environment subset, and Git commit info

This replaces “dump everything into `figures/`” so that results are traceable and reproducible.

### `experiments/python_scripts/exp3/exp3_toy_problems.py`
Role: Method of Manufactured Solutions — validate the DRM `BetaNet` architecture and the autograd-based spectral hyperbolicity penalty on two self-contained toy problems (no external dataset required). Defines a synthetic target \(\beta_{\mathrm{target}}(\hat Q)=0.1\exp(-\hat Q^2/2)+10^{-4}\) and trains against the analytically generated closing flux.

- **Toy Problem 2 (1D Slice):** Fix \(\hat R=5\), train on \(\hat Q\in[-2.5,2.5]\) (filtered to realizability). Two-panel figure: flux fit + \(\beta\) recovery.
- **Toy Problem 1 (2D Manufactured Solution):** Train on a \((\hat Q,\hat R)\) meshgrid inside the realizability domain. Three-panel heatmap: target \(\beta\), learned \(\beta\), spectral penalty map \(\mathcal P(\theta)\).

CLI: `python3 experiments/python_scripts/exp3/exp3_toy_problems.py [--gamma 1.0] [--lr 1e-3] [--n-epochs 2000] [--batch-size 256] [--hidden-dims 32 32] [--n-grid-2d 100] [--seed 42] [--device cpu]`

Outputs: `data/exp3_toy_problems/<timestamp>/` with:
- `model_toy2_1d.pt`, `model_toy1_2d.pt`
- `history_toy2_1d.json`, `history_toy1_2d.json`
- `figures/toy2_1d_slice.png/.pdf`
- `figures/toy1_2d_manufactured.png/.pdf`
- `figures/training_dynamics.png/.pdf`
- `summary.json`, `run_metadata.json`, `command.txt`, `run.log`

### `experiments/python_scripts/shock_structure/run_collisional_two_stream.py`
Role: Collisional two-stream test with static \(\beta\)-closure (Schaerer & Torrilhon 2015-style sweep). This script demonstrates the physical shock fronts propagating at speed \(v_0 + c_s \approx 4.5\), validating the locations where the neural network (Exp2) learns to place its regularization "walls" (\(\beta_\theta\) peaks). Archives `x.npy`, `u_final.npy`, figures, and metadata under `data/shock_structure_runs/two_stream_collisional/<timestamp>/`.

CLI: `python3 run_collisional_two_stream.py` (see script for optional flags)

Required packages: `torch`, `numpy`, `matplotlib`, `learning_moments_kinetics`
